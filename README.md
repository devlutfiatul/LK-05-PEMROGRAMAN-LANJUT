# LK-05-PEMROGRAMAN-LANJUT

Interface (MedicalRecord, Versioned, Confidential)
Ketiga interface ini berfungsi sebagai kontrak yang harus dipenuhi oleh setiap class data pasien. MedicalRecord memastikan setiap objek memiliki identitas unik melalui getPatientId(). Versioned digunakan untuk menandai versi data, sehingga sistem bisa membedakan antara format lama (V1) dan baru (V2). Sementara itu, Confidential menjadi bagian terpenting karena mengatur aspek keamanan data, yaitu menentukan level keamanan (getSecurityLevel()) dan bagaimana data ditampilkan berdasarkan level akses (getMaskedData(int clearanceLevel)). Dengan adanya interface ini, semua class turunan dipaksa memiliki struktur yang konsisten sehingga bisa diproses secara umum oleh sistem tanpa perlu mengetahui tipe spesifiknya.

Class PatientProfileV1
Class ini merepresentasikan versi awal data pasien yang masih sederhana, hanya berisi patientId, name, dan ssn. Class ini mengimplementasikan semua interface sehingga bisa digunakan dalam sistem yang sama dengan versi lain. Fungsi utama class ini adalah menyediakan data pasien serta mengatur masking sederhana melalui method getMaskedData(). Pada level akses rendah, SSN akan disembunyikan, sedangkan pada level akses yang cukup tinggi, seluruh data akan ditampilkan. Hubungannya dengan class lain adalah sebagai salah satu tipe data yang diproses oleh IntegrationGateway, sehingga walaupun sederhana, tetap mengikuti aturan keamanan yang sama.

Class PatientProfileV2
Class ini adalah pengembangan dari V1 dengan tambahan data yang lebih kompleks seperti allergyHistory dan medicalHistory. Karena datanya lebih sensitif, level keamanannya lebih tinggi dibanding V1. Method getMaskedData(int clearanceLevel) di class ini lebih kompleks karena menerapkan masking bertingkat: pada level 1 semua data sensitif disembunyikan, pada level 2 sebagian data dibuka (allergy dan riwayat penyakit), dan pada level 3 semua data ditampilkan. Class ini menunjukkan konsep evolusi data dalam sistem, tetapi tetap kompatibel karena mengikuti interface yang sama. Hubungannya dengan class lain sangat penting karena dapat diproses oleh IntegrationGateway tanpa perubahan kode, berkat polymorphism.

Class SecureResponse<T>
Class ini berfungsi sebagai pembungkus hasil yang dikembalikan oleh sistem. Ia menyimpan dua hal: data hasil (data) dan status apakah data tersebut sudah di-mask atau tidak (masked). Tujuannya adalah memisahkan antara isi data dan metadata keamanan, sehingga bagian lain dari sistem bisa mengetahui apakah informasi yang diterima sudah dibatasi atau masih asli. Class ini digunakan oleh IntegrationGateway sebagai output utama, sehingga semua hasil akses data menjadi konsisten formatnya.

Class IntegrationGateway<T>
Class ini merupakan inti dari sistem karena bertugas mengatur proses pengambilan data pasien dengan mempertimbangkan keamanan. Class ini menggunakan generic dengan batasan (T extends MedicalRecord & Versioned & Confidential) sehingga hanya objek yang memenuhi ketiga interface tersebut yang bisa diproses. Alur kerjanya adalah: pertama, mencocokkan patientId; jika tidak cocok, sistem mengembalikan pesan bahwa data tidak ditemukan. Jika cocok, sistem akan membandingkan level akses pengguna (clearanceLevel) dengan tingkat keamanan data (getSecurityLevel()). Jika akses tidak cukup, maka gateway akan memanggil getMaskedData() dari objek tersebut. Jika akses mencukupi, data asli ditampilkan melalui toString(). Karena semua class sudah menyediakan method masking sendiri, gateway tidak perlu menggunakan instanceof, sehingga desain menjadi lebih bersih dan fleksibel.

Class MainSimulation
Class ini berfungsi sebagai simulasi atau driver program untuk menguji seluruh sistem. Di dalamnya dibuat objek PatientProfileV1 dan PatientProfileV2, lalu masing-masing dimasukkan ke dalam IntegrationGateway. Selanjutnya dilakukan beberapa percobaan akses dengan level berbeda (1, 2, dan 3) untuk melihat bagaimana sistem merespons. Dari sini terlihat alur lengkap sistem: data dibuat → dimasukkan ke gateway → diproses berdasarkan level akses → hasil dikembalikan dalam bentuk SecureResponse. Class ini juga menunjukkan bagaimana satu gateway bisa menangani berbagai versi data tanpa perlu perubahan logika.
